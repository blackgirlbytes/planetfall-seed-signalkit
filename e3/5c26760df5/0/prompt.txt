Level 1: First Memories
Implemented in src/islandView.js.
The player lands on the island and finds three memory artifacts. For each one, they use a terminal flow:
text



git add -> git commit -> y to link Entire checkpoint

A committed memory freezes in ice. A checkpointed memory becomes safely banked. After all three are linked, the player must run:
bash



entire checkpoint list

before the countdown hits 0:00. This teaches: commits freeze work, checkpoints preserve/recover the context behind it.
Level 2: The Drone Bay
Implemented in src/droneBayView.js.
After Level 1, the ship’s drone bay wakes up. There are five broken ship systems and five drones. The player can send drones to different systems while one countdown runs.
Loop:
text



send drone -> drone works in parallel -> checkpoint appears -> explain checkpoint -> ADD TO SHIP

At a finished site, the terminal is prefilled with:
bash



entire checkpoint explain <id>

The player only presses Enter; they do not manually type checkpoint IDs. The explanation reveals what the drone actually did, then ADD TO SHIP/Y integrates the upgrade. After all five systems are online, the player types:
bash



entire dispatch

to generate the final day report.
The real lesson is not exactly “delegate review accept.” It is more like:
text



parallel agents can work while you do other things;
checkpoints let you understand work you did not watch;
dispatch summarizes the burst of work afterward.
